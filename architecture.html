项目简介

Muxy 是一款基于 SwiftUI + AppKit 的 macOS 终端应用，使用 libghostty 提供核心终端功能，并通过本地 WebSocket（MuxyServer）提供移动设备集成与远程控制能力。

核心模块

1) Muxy (可执行)

路径: Muxy/ — 主应用入口，包含 UI、App 入口以及资源

  • MuxyApp.swift — 应用入口与生命周期
  • Resources/ — 主题、terminfo 与运行时需要的静态资源
  • Views/ — SwiftUI 视图层（tabs、editor、statusbar 等）

2) MuxyShared (库)

路径: MuxyShared/ — DTO、协议与共享类型

  • 定义了模块间通信协议 (MuxyProtocol.swift)
  • 数据结构: WorkspaceDTO、ProjectDTO、VCSStatusDTO 等

3) MuxyServer (库)

路径: MuxyServer/ — 本地 WebSocket 服务，处理移动端连接与客户端控制

  • MuxyRemoteServer.swift — 服务实现
  • ClientConnection.swift — 单个连接的抽象

4) GhosttyKit (C wrapper)

路径: GhosttyKit/ — libghostty 的 C 桥接层，链接到预编译 GhosttyKit.xcframework

  • ghostty.h / GhosttyKit.c — 辅助导出给 Swift 的接口
  • module.modulemap — Swift Package 的 C 模块映射
  • GhosttyKit.xcframework — 预编译静态库（由 scripts/setup.sh 下载）

5) Tests

路径: Tests/MuxyTests — 单元与集成测试目标

依赖与链接

  • 第三方库 (Package.swift): Sparkle@2.9.1 (自动更新/升级)，Yams (YAML 解析)
  • 本机框架: AppKit, Carbon, CoreGraphics, CoreText, Foundation, IOKit, Metal, MetalKit, QuartzCore
  • 原生库: C++ 标准库 (链接 c++)
  • 外部资源: GhosttyKit.xcframework（平台二进制），Muxy/Resources 中的静态文件

运行与构建

# 准备
scripts/setup.sh    # 下载 GhosttyKit.xcframework

# 构建
swift build

# 运行
swift run Muxy

# 检查（格式化、lint、构建、测试）
scripts/checks.sh

数据持久化

  • Project 列表: ~/Library/Application Support/Muxy/projects.json
  • Ghostty 配置: ~/.config/ghostty/config

关键文件索引

  • Package.swift — Swift Package 配置（targets、依赖、链接设置）
  • CLAUDE.md — 本仓库的开发指南与注意事项
  • MuxyApp.swift — 应用入口
  • MuxyRemoteServer.swift — 远程服务实现
  • ghostty.h / GhosttyKit.c — C 桥接层
  • scripts/setup.sh — 下载与准备二进制依赖

架构要点与注意事项

  • 混合 UI 模式: SwiftUI + AppKit（确保在需要时使用 AppKit 的特性）
  • NSViewRepresentable 陷阱: 禁止在 makeNSView 中返回复用的 NSView（见 CLAUDE.md）
  • 资源管理: 主题/terminfo/rg 等资源需随发布包一起分发
  • 依赖二进制: GhosttyKit.xcframework 是必需的本地二进制依赖，由 setup.sh 下载，不应提交到 git
