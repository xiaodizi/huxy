## 推荐：AI 开发者的终极生产力终端

在原版 Muxy 的基础上，本 Fork 针对 AI 开发场景做了大量增强，尤其适合深度依赖 AI 编程助手的开发者和团队。

### 🤖 多 AI 编程助手一站集成
- 支持 Claude Code、Codex、Copilot、Amp、Z.ai、MiniMax、Kimi、Factory 等主流 AI 工具，随时切换，满足不同项目需求。
- 侧边栏实时显示各 AI 工具的用量配额，API 调用一目了然，避免额度超支。
- Claude Code 集成，AI 代码补全和对话体验无缝嵌入终端与编辑器。
- 支持 Cursor IDE 的 AI 使用追踪，方便统计和分析 AI 辅助开发效率。

### 🛠 AI 体验专属优化
- 编辑器扩展架构，支持多语言语法高亮、Markdown 实时预览、滚动同步，AI 生成内容体验极佳。
- 丰富快捷键和命令面板，AI 相关操作一键直达。
- 终端与文件树增强，适合 AI 生成代码的版本管理、对比和协作。

### 适用场景
- AI 辅助编码、自动化脚本生成、代码审查、团队协作等。

## 🏗 关键架构变更

### 工作区状态管理
- **AppState** (@Observable) - 根状态管理器，项目→标签→分割面板树（架构文档第 39 行）
- **WorkspaceReducer** - 工作区状态的纯函数 reducer（架构文档第 39 行）
- **NavigationHistory** - 导航历史栈，支持返回/前进（架构文档第 389-410 行）

### 数据持久化
- **ProjectStore** - 项目持久化到 `~/Library/Application Support/Muxy/projects.json`（架构文档第 31 行）
- **WorkspacePersistence** - 工作区布局持久化（架构文档第 120 行）
- **WorktreePersistence** - 每项目工作树持久化（架构文档第 117 行）
- **KeyBindingPersistence** - 快捷键持久化（架构文档第 109 行）
- **CommandShortcutStore** - 自定义命令快捷键持久化（架构文档第 110 行）

### 终端架构
- **GhosttyKit.xcframework** - 预编译的 libghostty 静态库（gitignored，通过 `scripts/setup.sh` 下载）
- **GhosttyTerminalNSView** - 包装 ghostty_surface_t 的 AppKit 视图（架构文档第 167 行）
- **TerminalViewRegistry** - 终端视图生命周期管理（架构文档第 170 行）
- **GhosttyRuntimeEventAdapter** - libghostty C 回调桥接（架构文档第 70 行）

### 文件操作
- **FileTreeService** - 惰性加载目录内容，尊重 .gitignore（架构文档第 103 行）
- **FileSystemOperations** - 创建/重命名/移动/复制/回收站操作（架构文档第 104 行）
- **FileSearchService** - 通过 `/usr/bin/find` 子进程快速打开文件（架构文档第 101 行）
- **TextSearchService** - 通过内置 ripgrep 进行文本搜索（架构文档第 102 行）

### 依赖项
- **Sparkle** - 自动更新框架
- **Yams** - YAML 解析（用于布局文件）
- **GhosttyKit.xcframework** - 终端仿真核心库

## 📋 构建与运行

```bash
# 安装依赖（首次）
scripts/setup.sh          # 下载 GhosttyKit.xcframework

# 开发构建
swift build               # 验证构建
swift run Muxy            # 运行应用

# 检查与测试
scripts/checks.sh         # 格式 → 语法检查 → 构建 → 测试
scripts/checks.sh --fix   # 自动修复格式和语法问题
```

工具版本锁定在 `.tool-versions`：
- swiftformat 0.60.1
- swiftlint 0.57.1
- xcode 16.4
